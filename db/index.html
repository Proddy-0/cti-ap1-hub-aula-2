Banco de Dados

O Banco de Dados é um sistema organizado para armazenar, gerenciar e recuperar informações de forma eficiente.
Para projetar um banco de dados antes de criá-lo, usamos o DER Diagrama Entidade-Relacionamento.
Ele representa visualmente as entidades, seus atributos e como elas se relacionam entre si.

No DER, as participações definem se uma entidade é obrigada ou não a participar de um relacionamento.
A participação total significa que toda instância da entidade precisa estar associada ao relacionamento representada por linha dupla.
Já a participação parcial significa que a associação é opcional representada por linha simples.

O DER também permite representar especializações, que é quando uma entidade se divide em subtipos mais específicos.
A especialização disjunta indica que uma instância só pode pertencer a um subtipo por vez por exemplo, uma pessoa só pode ser física ou jurídica.
Já a especialização overlapping (sobreposição) permite que uma instância pertença a mais de um subtipo ao mesmo tempo.

Entender a participação total, parcial, disjunta e overlapping é essencial para modelar um banco de dados corretamente.
Essas definições feitas no DER refletem diretamente as regras do mundo real que o sistema precisa respeitar.
Um DER bem construído evita erros na estrutura do banco e facilita toda a etapa de implementação.
